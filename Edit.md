Firefly（流萤）Astro 博客模板个性化改造完整指南
你 Fork 的 CuteLeaf/Firefly 是基于 Astro + Tailwind CSS 打造的高颜值静态博客模板，支持主题色、布局、组件、内容全方位自定义。下面从环境准备、基础配置、样式美化、布局改造、内容管理、进阶定制、部署上线一步步教你改成专属风格，全程适配 Fork 后的仓库使用。
一、前期准备：本地运行项目（必做）
1. 环境依赖
该模板要求：Node.js ≥ 22、pnpm ≥ 9，提前安装工具：
安装 Node.js：官网下载 LTS 新版，安装后终端执行 node -v 验证版本。
安装 pnpm：终端执行 npm install -g pnpm，执行 pnpm -v 验证。
安装 Git：用于代码拉取、提交和同步仓库。
2. 拉取 Fork 后的代码到本地
打开你的 GitHub 账号，找到你 Fork 后的 Firefly 仓库。
终端执行命令克隆代码（替换为你自己的仓库地址）：
bash
运行
git clone https://github.com/你的GitHub用户名/Firefly.git
cd Firefly
安装项目依赖：
bash
运行
pnpm install
启动本地开发服务（热更新，修改代码实时预览）：
bash
运行
pnpm dev
浏览器访问 http://localhost:4321，即可看到博客默认页面，后续所有修改都能实时生效。
二、第一步：基础信息定制（零代码，快速改身份）
模板所有站点基础配置集中在 src/config/ 目录下，无需修改代码文件，直接编辑配置文件即可完成个性化，优先级最高。
1. 站点核心信息（名称、简介、作者、SEO）
打开 src/config/site.ts（或 config.ts），修改以下内容：
title：博客首页标题、浏览器标签名
description：网站描述（用于搜索引擎、社交分享）
author / avatar：作者名称、头像（替换 public 目录下的头像图片即可）
url：填写你最终的博客域名（部署后生效）
social：微信、GitHub、知乎、邮箱等社交链接，按需增删
2. 导航栏 & 页脚配置
导航栏：编辑 src/config/nav.ts，自定义顶部菜单、导航链接、分类，支持新增 / 删除栏目、修改跳转地址。
页脚：编辑 src/config/footer.ts，修改版权信息、备案号、底部文案，模板支持 HTML 内容注入，可自定义文字、图标。
3. 基础功能开关
src/config/feature.ts 控制全局功能：
开启 / 关闭搜索功能（基于 Pagefind 全文搜索）
开启 / 关闭暗色 / 亮色模式（支持跟随系统切换）
侧边栏配置：单侧边栏、双侧边栏切换，开启 / 关闭日历、文章统计、目录等组件
页面动画、过渡效果开关
三、第二步：视觉风格改造（配色、字体、背景）
该模板基于 Tailwind CSS，视觉定制门槛极低，分「简易配置」和「深度样式修改」两种方式。
1. 一键修改主题色（推荐新手）
模板原生支持全局主题色相调节：
找到配置文件中 themeColor 相关字段，输入十六进制色值（如 #ff6700 橙色、#165DFF 蓝色）。
修改后本地页面实时变色，按钮、链接、高亮文字、标签等全局同步更新。
额外配置：亮色 / 暗色模式独立配色，分别设置两套主题色。
2. 背景与壁纸风格
支持三种背景模式，在配置中直接切换：
纯色背景：自定义全局背景色
横幅壁纸：首页顶部大图横幅，替换 public 目录下壁纸图片即可
全屏透明壁纸：毛玻璃透明风格，适合简约风博客
3. 字体自定义
全局字体：在配置文件中修改 font 字段，模板内置多款开源字体，直接选择即可。
引入自定义字体：将字体文件放入 public/fonts/，再修改 src/styles/global.css 全局样式文件，替换字体栈。
4. 深度 CSS 样式修改（进阶）
如需微调间距、圆角、边框、hover 效果：
全局样式文件：src/styles/global.css，所有全局样式、CSS 变量都在这里。
示例（修改默认圆角、卡片阴影）：
css
:root {
  --radius: 12px; /* 全局圆角大小 */
  --shadow: 0 2px 12px rgba(0,0,0,0.08); /* 卡片阴影 */
}
组件样式：页面模块（卡片、按钮、评论框）样式分散在 src/components/ 对应 .astro 文件中，可直接修改 Tailwind 类名调整样式。
四、第三步：布局结构改造（页面排版、侧边栏、文章样式）
Firefly 主打高自由度布局，所有布局均可通过配置文件 + 组件文件修改，无需重构代码。
1. 首页文章布局切换
在配置中一键切换 3 种主流布局：
列表布局：传统博客单列文章，适合文字为主的博客
网格布局：多列卡片，适合图文、技术博客
瀑布流布局：错落卡片，适合摄影、生活类博客
2. 侧边栏定制
布局选择：单侧边栏（左 / 右）、双侧边栏，根据喜好切换。
组件增删：侧边栏支持文章目录、最新文章、标签云、日历、访问统计等组件，在配置中开关即可。
位置调整：如需修改侧边栏宽度、位置，编辑 src/layouts/ 下的全局布局文件。
3. 独立页面改造（关于页、归档、标签页）
固定页面文件位置：src/pages/，每个 .astro 文件对应一个独立页面。
index.astro：首页
about.astro：关于我页面（修改个人简介、经历、技能）
archive.astro：文章归档页
tags.astro：标签页
自定义页面内容：直接编辑页面内文本、图片、模块，支持嵌入 HTML、图标。
五、第四步：内容管理（写文章、分类、标签）
博客核心是内容，所有文章统一使用 Markdown/MDX 编写，规则如下：
文章存放目录：src/content/posts/，所有博文都新建 .md 或 .mdx 文件。
文章头部配置（Frontmatter）：每篇文章顶部添加基础信息，示例：
markdown
---
title: "文章标题"
date: "2026-06-15"  # 发布日期
tags: ["技术", "Astro"] # 标签
categories: "前端" # 分类
excerpt: "文章摘要" # 首页展示摘要
cover: "/images/cover.jpg" # 文章封面图（图片放 public/images）
draft: false # 是否为草稿（true 则不展示）
---
这里写正文内容（支持标准 Markdown 语法、代码块、表格、图片）
图片管理：所有静态图片（封面、配图、头像）统一放入项目根目录 public/ 文件夹，引用路径直接写 /图片路径 即可。
六、第五步：高阶定制（组件、功能、交互）
适合有前端基础，想深度改造的用户：
1. 组件自定义
所有可复用模块（导航栏、页脚、卡片、评论、搜索框）都在 src/components/ 目录：
Navbar.astro：导航栏，可替换 Logo、修改菜单样式、新增图标按钮。
Footer.astro：页脚，自定义链接、图标、文案。
PostCard.astro：首页文章卡片，修改卡片样式、封面展示逻辑。
2. 评论系统接入
模板预留评论接口，可按需接入主流评论工具：
Giscus（基于 GitHub 讨论，推荐）、Waline、Utterances 等，按照对应文档在组件中引入代码即可。
3. 多语言、国际化
模板原生支持 i18n 多语言，在配置中切换简体中文、繁体、英文等，也可自定义翻译文本。
4. 插件与功能扩展
Astro 生态丰富，可新增功能：
代码高亮、行号显示（模板默认支持，可修改主题）
图片懒加载、灯箱预览
阅读时长、点赞、访客统计
七、第六步：打包、提交代码 & 部署上线
本地调试完成后，打包静态文件并部署到网络，让全网可访问。
1. 打包生产文件
终端执行打包命令，会在根目录生成 dist 静态文件夹（最终上线文件）：
bash
运行
pnpm build
执行 pnpm preview 可本地预览打包后的线上效果。
2. 同步代码到你的 GitHub 仓库
将所有修改提交到你 Fork 的远程仓库：
bash
运行
# 查看修改文件
git status
# 暂存所有修改
git add .
# 提交备注
git commit -m "个性化改造完成：修改主题色+首页布局"
# 推送到远程仓库
git push
3. 主流部署方案（任选其一）
GitHub Pages（免费）
配合 GitHub Actions 实现自动部署：推送代码后自动打包、上线，参考 Astro 官方部署文档配置工作流文件 .github/workflows/deploy.yml 即可。
Cloudflare Pages（推荐，国内访问稳定）
绑定你的 GitHub Fork 仓库，开启自动部署，速度快、无需额外配置。
Vercel / Netlify
导入 GitHub 仓库，平台自动识别 Astro 框架，一键部署，自带 HTTPS。
自有服务器 / 虚拟主机：直接上传 dist 文件夹内所有文件。
八、维护与同步（Fork 仓库必备）
原作者会持续更新 Firefly 模板，你可以同步上游更新，保留自己的修改：
添加上游原仓库地址：
bash
运行
git remote add upstream https://github.com/CuteLeaf/Firefly.git
拉取原仓库最新代码：
bash
运行
git pull upstream main
解决少量代码冲突后，重新提交、部署即可。
九、避坑小贴士
图片路径：所有静态资源必须放在 public 目录，不要放在 src 内。
依赖报错：若 pnpm install 失败，检查 Node.js/pnpm 版本是否达标。
草稿文章：draft: true 的文章线上不会展示，发布时记得改为 false。
样式不生效：修改全局样式后，重启 pnpm dev 或清空浏览器缓存。
按照以上步骤，你可以从基础信息 → 视觉风格 → 布局结构 → 内容写作 → 部署上线完整打造专属博客。新手建议先从「配置文件」入手改基础信息和配色，熟悉后再调整布局和组件。
